# Workspace Layout

```
MyAgency/
  .multigent/
    agency.yaml              # workspace metadata
    inbox.yaml               # human task-confirmation inbox (auto-managed)
    inbox.md                 # human-readable inbox (auto-generated)
    messages.yaml            # async messages delivered to human

  .gitignore                 # auto-generated on agency create

  agency-prompt.md           # agency-wide context

  teams/
    engineering/
      team.yaml
      prompt.md
      roles/
        developer/
          role.yaml          # skills[], setup (dirs/files to create)
          prompt.md          # role-specific context layer

  skills/                    # no built-ins — define only what you need
    github-push-relay/
      SKILL.md               # YAML frontmatter + Markdown instructions
      git-push-github.sh     # bundled file, chmod+x preserved

  project-blueprints/        # project templates packaged with the agency template
    default.yaml             # declares agents, heartbeats, and playbooks

  projects/
    my-api/
      project.yaml           # declarative: agents + heartbeats + crons + playbooks
      prompt.md              # project-specific context
      agents/
        dev/
          CLAUDE.md              # merged context (@imports all layers)
          .multigent/
            agent.yaml           # agent metadata
            context/
              agency.md         # agency context layer
              role-<team>-<role>.md
              project-<project>.md
              wakeup.md          # playbook (installed by project apply)
            tasks.yaml           # active task queue
            tasks_archive.yaml   # completed tasks
            heartbeat.yaml       # wakeup config (set by project apply)
            crons.yaml          # scheduled tasks (set by project apply)
            messages.yaml       # async messages delivered to this agent
            runs/               # execution logs
          .claude/skills/       # deployed skill files
```

## Key files explained

### `agency-prompt.md`
Agency-wide instructions injected into every agent's context. Define your company values, coding standards, communication norms, etc.

### `teams/<team>/prompt.md`
Team-level context. For example, `engineering/prompt.md` might define code review standards, branch naming conventions, and tech stack choices.

### `teams/<team>/roles/<role>/prompt.md`
Role-specific instructions. A `developer` role might include how to write commit messages; a `pm` role might include how to prioritise issues.

### `skills/<skill>/SKILL.md`
Reusable capability definitions deployed into agent working directories. Put metadata in YAML frontmatter and instructions in the Markdown body. Use `{{SKILL_DIR}}` to reference co-located scripts and files.

### `project-blueprints/<name>.yaml`
Declarative project template defining which agents to hire, their heartbeat schedules, sandbox settings, and which playbook each agent uses.

### `projects/<project>/agents/<agent>/CLAUDE.md`
The fully assembled context file for this agent. Auto-generated by `hire` / `sync`. Do not edit manually — edit the source layers instead.

### `projects/<project>/agents/<agent>/.multigent/tasks.yaml`
The active task queue for this agent. The scheduler reads this file to find the next task to execute.

### `projects/<project>/agents/<agent>/.multigent/heartbeat.yaml`
Heartbeat configuration: interval, active_hours, active_days, wakeup_prompt path.

## Context merging

When you run `hire` (or `project apply`), multigent merges the context chain:

```
agency-prompt.md
  + teams/<team>/prompt.md
  + teams/<team>/roles/<role>/prompt.md
  + projects/<project>/prompt.md
  + skills (all skills bound to team + role)
```

The result is written as a single context file (`CLAUDE.md`, `AGENTS.md`, etc.) in the agent's working directory. Change any layer and run `multigent sync` to propagate — only agents affected by the changed file are updated.
